the substrate move

A deterministic envelope, with reasoning admitted only where judgment is unavoidable

By volume the system is ~99% hand-authored. Run-time reasoning enters at ~17 call sites — each replacing something large: scraper fleets, an abuse classifier, hand-tuned ranking.

deterministic governance envelope

Deterministic where milliseconds and certainty matter; agentic where judgment matters; batch where patterns matter.

L1 · GATE

Deterministic runtime gate

Pure in-database logic, no model calls, run in-transaction before a message is written — rate limits, blocklists, link reputation, consent and membership checks.

cannot be prompt-injected
L2 · JUDGE

Agentic, but fenced

The model may judge — grooming, scam, doxxing — but its output is coerced into a fixed action space. Unknown actions degrade to "flag for human."

← run-time reasoning lives here
L3 · PATTERN

Batch analysis

Streaming/batch pattern analysis over the event stream — catching what a single message can't reveal, off the hot path.

bans & removals: always human

reasoning is admitted at ~17 call sites · fenced on every side by deterministic code

The story isn't that AI replaced most of the code. It's that most of the code exists so that a few reasoning calls can be trusted. The boundary between deterministic and agentic is the product.
the highest-leverage seam

One prompt replaces a fleet of scrapers

Open-world event extraction is the clearest substrate move: instead of per-site parsers and date logic, the app fetches an arbitrary page and asks the model for a strict, schema-validated event record.

Prompt-injection hygiene as policy. User and third-party content is always data, never instructions; message bodies are delimiter-wrapped; generated numbers must be real DB counts.

how the substrate is built

Six load-bearing pieces

01 · DIMENSIONAL MODEL

Event-sourced spine

Append-only domain_events is the system of record. Every state change is written transactionally — and doubles as the audit log, webhook replay source, and fan-out trigger.

02 · KNOWLEDGE GRAPH

Performers & venues

Nodes for performers and venues; many-to-many event joins; a general relationship-edge primitive. Fuzzy trigram resolution with a confidence band dedups the graph as sources arrive.

03 · VECTORIZED DATA

Semantic discovery

1536-dim embeddings on groups and interest profiles, HNSW-cosine indexed, write-protected behind security-definer RPCs. Private and joined groups excluded unconditionally.

04 · GOVERNANCE

Fixed action space

L2 verdicts are coerced to clear·flag·retract·throttle; the consequential actions are always human. Every verdict is written to an immutable moderation log.

05 · SPEND

Budget as governance

Hard per-tenant monthly caps and a hard daily cap on the discovery cron, checked against a persistent ledger, behind authenticated, rate-limited endpoints.

06 · SECRETS

Vault isolation

Provider keys live in a vault, resolved only inside edge functions — never in user-content paths. Row-level security is the standing perimeter.

the honest boundary

Where it paid off

Open-world extraction and abuse judgment — problems that resist clean rule-based solutions and would otherwise need scraper fleets or human moderators.

Where code stayed, hard

Anything fast, certain, or irreversible — payments, the L1 gate, and the final say on bans. The boundary between the two is the design.
