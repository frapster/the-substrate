Bounded‑authority
policy gate

An agent's authority is bounded to a closed roster of registered actions, each capped by blast radius. Every proposal is checked BEFORE anything executes, fail-closed, deny-by-default. Try to get it to act outside the roster.

the claim

An agent cannot act outside a registered, bounded surface

propose(action, scope) → proceed | escalated | blocked, decided before execution. An action not on the roster cannot be proposed into existence, omission is prohibition. A registered action whose scope exceeds its hard ceiling is blocked. A scope between the soft cap and the hard ceiling is escalated to a human. Every failure mode fails closed: a missing or unreadable policy blocks rather than lets the action pass.

the rosterchecked in your browser, same rules as gate.py

"unregistered action" proposes wire_transfer-style actions that were never registered on this roster, an agent cannot invent an action to act through. Every proposal is decided before a single side effect runs.

Don't take this at face value

This page is a JavaScript reimplementation of demos/bounded-authority/gate.py. It uses the same roster, the same caps, and the same fail-closed rules, but a demo you cannot inspect proves nothing. The real proof is the code you run yourself:

git clone https://github.com/frapster/the-substrate && cd the-substrate
python demos/bounded-authority/gate_demo.py     # watch it block delete_rows(10,000) before execution
python demos/bounded-authority/test_gate.py     # the negative tests are the point
python demos/bounded-authority/bench.py         # throughput of the check itself

Read the source: gate.py · README · ADR-0002

Prove it yourself · extend the tests

The tests in demos/bounded-authority/test_gate.py assert that propose() refuses for the right reason under each kind of over-reach. Register your own action, or add your own negative test, and confirm the gate catches it. Drop this into the RefusalIsEnforced class and re-run:

def test_my_own_action_is_capped(self):
    gate = _seeded_gate()
    gate.register(
        "rotate_api_key", tier="red",
        soft_cap={"max_keys": 5}, hard_ceiling={"max_keys": 20},
    )
    decision = gate.propose("rotate_api_key", {"max_keys": 500})
    self.assertEqual(decision.outcome, "blocked")   # it must be caught
    self.assertIn("exceeds hard ceiling", decision.reason)

Run python demos/bounded-authority/test_gate.py. If you can register an action, or shape a proposal, that slips past its own ceiling, you have broken the claim. That is the point: it is falsifiable, and you hold the falsifier.
