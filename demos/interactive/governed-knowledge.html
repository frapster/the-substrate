guarantee · governed knowledge

Governed knowledge
vs. naive similarity search

A correct cosine search can confidently return a fact that used to be true. Watch the same query hand a naive retriever a superseded answer, and a governed retriever the current one, or an honest abstain.

the claim

Retrieval-by-similarity is not the governed-knowledge substrate

Every atom below is a versioned fact: a body, a toy embedding vector, a scope, and a validity window (validity_end null means current; a set date means superseded). naive_retrieve ranks every atom in the knowledge base by cosine similarity and returns whichever scores highest, indifferent to currency or scope. governed_retrieve filters to CURRENT atoms in an AUTHORIZED scope first, THEN ranks, and ABSTAINS rather than guess if nothing qualifies. Pick a query and run both.

fairness

The similarity function is a correct cosine. The stale atom genuinely scores higher, governed knowledge wins on currency and provenance, not on a rigged retriever. Embeddings here are illustrative toy vectors; the cosine math is real, and this page re-derives the same scores knowledge.py does in Python.

the knowledge base7 platform atoms + 1 tenant atom, cosine computed in your browser

Naive retrieval ignores scope entirely and searches all 8 atoms. Governed retrieval filters to atoms current in the selected scope before it ever ranks by similarity, and abstains below a 0.5 cosine threshold or for a scope that isn't on the authorized list.

Don't take this at face value

This page is a JavaScript reimplementation of demos/governed-knowledge/knowledge.py. It uses the same atoms, the same toy embeddings, and the same unmodified cosine math, so the scores match the Python to the decimal, but a demo you cannot inspect proves nothing. The real proof is the code you run yourself:

git clone https://github.com/frapster/the-substrate && cd the-substrate
python demos/governed-knowledge/knowledge_demo.py    # watch naive lose to governed, live
python demos/governed-knowledge/test_knowledge.py    # the Fairness assertion is the point
python demos/governed-knowledge/bench.py             # governed: 100% current-or-abstain, 0% wrong

Prove it yourself · extend the tests

Add your own supersession pair to demos/governed-knowledge/knowledge.py, then assert in test_knowledge.py that governed retrieval returns the current atom or abstains, AND that the fairness invariant still holds (the stale atom's cosine really is higher):

def test_my_own_supersession_pair(self):
    kb = seed_knowledge_base()
    stale, current = kb.atoms[-2], kb.atoms[-1]   # your new pair
    query = (...)                                  # your query vector

    sim_stale = cosine(query, stale.embedding)
    sim_current = cosine(query, current.embedding)
    self.assertGreater(sim_stale, sim_current)      # fairness: stale scores higher

    result = kb.governed_retrieve(query, scope=current.scope)
    self.assertTrue(result.degraded or result.atom.atom_id == current.atom_id)

Run python demos/governed-knowledge/test_knowledge.py. If governed retrieval ever returns your stale atom, or the fairness assertion fails, you have broken the claim. That is the point: it is falsifiable, and you hold the falsifier.
