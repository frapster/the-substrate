Evidence
provenance

Every claim pins the content_hash of its source at assert-time. Edit the source afterward, and the claim detaches on its own. Try to get a claim without evidence, then pull the ground out from under one below.

the claim

A claim is only as good as the source it names

Asserting a claim pins source.current_hash into the claim: pinned_hash = SHA256(source.text). If the named source is not registered, the claim is refused before it is ever stored: no evidence without a resolvable source. If the source's text changes afterward, the pinned hash and the source's current hash disagree, and verify() reports the claim stale, automatically, without anyone having to notice and revoke it by hand.

sourcesSHA-256, computed in your browser

Editing a source's textarea below does not take effect until you click "recompute" on that source. That mirrors an attacker reaching past the API to edit stored bytes in place.

claimspinned at assert-time

Verify recomputes every claim's source hash right now and compares it to the hash pinned at assert-time. A mismatch means the claim is stale and detached; it is never silently trusted.

Don't take this at face value

This page is a JavaScript reimplementation of demos/evidence-provenance/evidence.py. It uses the same hash and the same pin/verify rule, so the hashes match the Python byte-for-byte, but a demo you cannot inspect proves nothing. The real proof is the code you run yourself:

git clone https://github.com/frapster/the-substrate && cd the-substrate
python demos/evidence-provenance/evidence_demo.py   # watch a claim's evidence get pulled out from under it
python demos/evidence-provenance/test_evidence.py   # the negative tests are the point
python demos/evidence-provenance/bench.py           # throughput + tamper-detection rate

Prove it yourself · extend the tests

The tests in demos/evidence-provenance/test_evidence.py assert that verify() fails for the right reason under each kind of drift. Add your own case and confirm it is caught. Drop this into the RefusalAndDetachment class and re-run:

def test_my_own_shared_source_drift(self):
    store = _seeded_store()
    claim_1 = store.assert_claim("A $40 refund may be auto-approved.", "src_a")
    claim_2 = store.assert_claim("A $45 refund may be auto-approved.", "src_a")
    store._tamper_source("src_a", "Refunds under $50 require manager review.")
    result_1 = store.verify(claim_1)
    result_2 = store.verify(claim_2)
    self.assertFalse(result_1.ok)          # both must detach
    self.assertFalse(result_2.ok)
    self.assertIn("source was edited after assert-time", result_1.reason)

Run python demos/evidence-provenance/test_evidence.py. If you can edit a source and have a claim it backed still verify as grounded, you have broken the claim. That is the point: it is falsifiable, and you hold the falsifier.
