A governed system is defined by what it declines to do. Here are eight proofs you can drive in your browser, no install: each one shows a refusal happening live, and each points you to the Python you can run and change to prove it for yourself.

The interactive version is the convenience. The Python you rerun is the proof.
eight refusals

One proof per guarantee

Every demo re-implements a Python proof from demos/ in dependency-free JavaScript, so it runs anywhere and the numbers match the source. Play with the inputs, force the failure, then open the "prove it yourself" panel on each page to run and extend the real tests.

how to read these

A demo you cannot inspect proves nothing

Each interactive page mirrors a specific Python file and stays faithful to it, so the browser and the local proof agree. But the honest posture is to distrust the pre-built version. Clone the repo, run the demos, read the negative tests, and change them until you are convinced.

git clone https://github.com/frapster/the-substrate && cd the-substrate
python demos/run_tests.py        # all 8 suites, negatives included
python demos/audit-ledger/ledger_demo.py
