guarantee · audited

Tamper-evident
audit ledger

Every governed decision writes a hash-chained entry. Edit one past row, and recomputation catches it. Try to forge history below.

the claim

Silent edits to committed history are impossible

Each row folds the previous row's hash into its own: hash = SHA256(prev_hash | index | canonical_json(body)). Change any past row and its hash no longer matches what the next row recorded, so verify() pinpoints the first break. Commit a decision, flip a past verdict, then verify.

the chainSHA-256, computed in your browser

The tamper button edits a stored row's verdict without recomputing its hash, exactly as someone with database access would. Verify recomputes the whole chain from genesis and reports the first row where stored and recomputed disagree.

Don't take this at face value

This page is a JavaScript reimplementation of demos/audit-ledger/ledger.py. It uses the same chaining rule, so the hashes match the Python byte-for-byte, but a demo you cannot inspect proves nothing. The real proof is the code you run yourself:

git clone https://github.com/frapster/the-substrate && cd the-substrate
python demos/audit-ledger/ledger_demo.py     # watch it catch a forged row
python demos/audit-ledger/test_ledger.py     # the negative tests are the point
python demos/audit-ledger/bench.py           # ~74k rows/sec verify, 200/200 tampers caught

Read the source: ledger.py · README · ADR-0001

Prove it yourself · extend the tests

The tests in demos/audit-ledger/test_ledger.py assert that verify() fails for the right reason under each kind of tampering. Add your own attack and confirm the chain catches it. Drop this into the TamperIsCaught class and re-run:

def test_my_own_tamper(self):
    ledger = _seed(6)
    # try YOUR attack: edit a body, swap two rows, delete one…
    edited = dict(ledger.rows[4].body)
    edited["amount_usd"] = 999999
    ledger._tamper_body(4, edited)      # hash left stale
    result = ledger.verify()
    self.assertFalse(result.ok)          # it must be caught
    self.assertEqual(result.broken_index, 4)

Run python demos/audit-ledger/test_ledger.py. If you can make a silent edit that verify() misses, you have broken the claim. That is the point: it is falsifiable, and you hold the falsifier.
